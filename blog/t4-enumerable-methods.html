Navigating your code with map

For instances in your coding where you need to navigate through some array or hash, you could certainly use a "for" statement. This would look something like this:

*Both images taken from here*

But what if you wanted to reuse the original data contained in the variable "arr"? Using a "for" statement and executing the code within will alter or change the original data. Luckily there's a solution for this.

Mapping

Making use of the "map" method will allow you to iterate over an array or hash and produce a new array with the results of your code. It would look something like this:
*Both images taken from here*

Map can be used alternately by calling "collect". These two methods work in the same way with the same syntax. There is one more way in which to invoke the "map" method, and this one is destructive. This means that by calling "map!" with the exclamation mark, the code that is in the block will iterate through each element and actually alter it's value. So, you can get the functionality of an "each" method by calling "map!"
